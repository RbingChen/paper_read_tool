内容理解

该文本包含两个消融实验结果图示（图25和图26）以及数据源选择标准说明。核心内容揭示：

  • 在增加训练资源的前提下，通过两种不同策略（增加DR或增大group size）提升rollout budget，均能在总训练时间不变的情况下改善模型测试性能
  • 数学领域选用NuminaMath-1.5数据集（89.6万数学问题），但需严格质量审查
  • 代码领域面临数据集质量不足的问题，测试CODE-RL、TACO、Eurus-RL等数据集效果不佳
  • 数据筛选首要标准是Verifiable（可验证性），排除无测试用例的代码问题及证明类数学问题

6.1 Data Source Selection and Preprocessing
6.1 数据源选择与预处理
For the math domain, we primarily focus on NuminaMath-1.5 [13], a comprehensive dataset containing 896K math problems drawn from widely used sources and advanced mathematical topics. Although the dataset is sufficiently large, its quality requires careful examination prior to use.
在数学领域，我们主要关注NuminaMath-1.5[13]数据集，这是一个包含89.6万数学问题的综合性数据集，涵盖广泛来源和高级数学主题。尽管该数据集规模足够大，但其质量在使用前需要仔细检查。
For the code domain, we find that data source options are more limited, and the overall difficulty of available datasets is generally low relative to the capabilities of current models. In our pilot studies, we experimented with several popular datasets – including CODE-RL [12], TACO [14], and the Eurus-RL collection [2] – in their original mixtures, but obtained unsatisfactory results.
在代码领域，我们发现数据源选择更为有限，且现有数据集的整体难度相对于当前模型的能力普遍较低。在初步研究中，我们尝试了多个流行数据集的原始混合——包括CODE-RL[12]、TACO[14]和Eurus-RL集合[2]——但获得的结果不尽人意。
Selection Criteria To select and curate high-quality data for RL, we adhere to the following general criteria for both data domains:
选择标准：为强化学习选择和整理高质量数据时，我们在两个数据领域均遵循以下通用标准：
1.Verifiable : We exclude problems that cannot be verified, such as proof-based problems and code problems lacking test cases.
1. 可验证性(Verifiable)：我们排除无法验证的问题，例如基于证明的问题和缺少测试用例的代码问题。

摘要总结

本文本核心内容包含：

  • 消融实验结果：增加训练资源时，通过提升rollout budget（采用增大DR或group size策略）可在同等训练时长下显著提升模型测试性能
  • 数据源选择：数学领域采用NuminaMath-1.5大规模数据集（需质量验证），代码领域现有数据集（如CODE-RL、TACO）难度不足且效果不佳
  • 筛选标准：首要准则为可验证性(Verifiable)，排除无法验证的问题类型（如无测试用例的代码问题）

术语识别

Ablation Experiments
消融实验：通过系统性移除模型组件或训练条件来评估各要素对性能贡献的实验方法
rollout budget
推演预算：强化学习中用于环境交互的轨迹采样资源总量限制
DR
动态范围/数据复用（具体根据上下文）：可能指动态调整采样范围或数据复用策略的参数
group size
组大小：并行训练中同时推演的实例数量，影响探索效率
NuminaMath-1.5
数学问题数据集：包含89.6万数学问题的综合数据集，覆盖基础到高级数学主题
CODE-RL
代码强化学习数据集：专为代码生成任务设计的强化学习训练数据集
TACO
代码优化数据集：通常包含代码转换/优化任务的基准数据集
Eurus-RL
强化学习集合：整合多种强化学习任务的数据集组合
Verifiable
可验证性：数据筛选核心标准，要求问题必须存在客观验证方法（如测试用例）
